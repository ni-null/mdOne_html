# Multi-language Support (i18n)

## Overview

MDOne supports multiple languages through JSON configuration files. Currently includes:

- **en** - English
- **zh-TW** - Traditional Chinese

## Language File Structure

Each language has two sections:

### 1. CLI Block (Developer-facing)

Strings used in console output and error messages:

```json
{
  "cli": {
    "app_title": "MDOne HTML Packager",
    "err_no_templates": "[ERROR] No templates found.",
    "err_no_markdown": "[ERROR] No markdown files found.",
    "success": "[OK] Done!",
    "processing": "Processing..."
  }
}
```

### 2. Template Block (UI strings)

Strings injected into generated HTML via placeholders:

```json
{
  "template": {
    "html_lang": "en",
    "page_subtitle": "Documentation",
    "loading": "Loading...",
    "footer_label": "Built on {BUILD_DATE}",
    "menu_open_title": "Open menu",
    "breadcrumb_docs": "Docs",
    "breadcrumb_index": "Home",
    "status_offline": "Offline Mode",
    "theme_light_label": "Light",
    "theme_dark_label": "Dark",
    "theme_toggle_title": "Toggle theme",
    "doc_count_suffix": "docs"
  }
}
```

## Using i18n in Templates

In `template.html`, use uppercase placeholder format:

```html
<html lang="{LANG}">
<head>
    <title>{TITLE}</title>
</head>
<body>
    <h2>{I18N_PAGE_SUBTITLE}</h2>
    <p>{I18N_LOADING}</p>
    <footer>{I18N_FOOTER_LABEL}</footer>
</body>
</html>
```

## Supported Placeholders

| Placeholder | Source | Description |
|------------|--------|-------------|
| `{I18N_HTML_LANG}` | `template.html_lang` | HTML lang attribute (e.g., "en", "zh-TW") |
| `{I18N_PAGE_SUBTITLE}` | `template.page_subtitle` | Page subtitle |
| `{I18N_LOADING}` | `template.loading` | Loading text |
| `{I18N_FOOTER_LABEL}` | `template.footer_label` | Footer text (supports `{BUILD_DATE}`) |
| `{I18N_MENU_OPEN_TITLE}` | `template.menu_open_title` | Menu button title attribute |
| `{I18N_BREADCRUMB_DOCS}` | `template.breadcrumb_docs` | Breadcrumb "Docs" label |
| `{I18N_BREADCRUMB_INDEX}` | `template.breadcrumb_index` | Breadcrumb home label |
| `{I18N_STATUS_OFFLINE}` | `template.status_offline` | Offline status label |
| `{I18N_THEME_LIGHT_LABEL}` | `template.theme_light_label` | Light theme button text |
| `{I18N_THEME_DARK_LABEL}` | `template.theme_dark_label` | Dark theme button text |
| `{I18N_THEME_TOGGLE_TITLE}` | `template.theme_toggle_title` | Theme toggle button title |
| `{I18N_DOC_COUNT_SUFFIX}` | `template.doc_count_suffix` | Suffix for document count |

## Adding a New Language

### Step 1: Create Language File

Copy `locales/en.json`:

```bash
cp locales/en.json locales/es.json
```

### Step 2: Translate

Edit `locales/es.json` and translate all `cli` and `template` values:

```json
{
  "cli": {
    "app_title": "Empaquetador HTML MDOne",
    "err_no_templates": "[ERROR] No se encontraron plantillas.",
    "success": "[OK] ¡Hecho!"
  },
  "template": {
    "html_lang": "es",
    "page_subtitle": "Documentación",
    "loading": "Cargando...",
    ...
  }
}
```

**Important:**
- Set `html_lang` to the correct language code
- Keep all JSON keys (don't add/remove keys)
- Clean JSON syntax (proper quotes, commas)

### Step 3: Use the Language

```bash
# Via .env
echo "LOCALE=es" >> .env
python main.py

# Or via command line
python main.py --locale es
```

## Translation Checklist

When adding a new language:

- [ ] File created: `locales/your_locale.json`
- [ ] All `cli` keys translated
- [ ] All `template` keys translated
- [ ] `template.html_lang` set to correct code (e.g., "fr", "de", "ja")
- [ ] JSON syntax valid (use a JSON linter)
- [ ] Footer date format preserved in translation
- [ ] Tested: `python main.py --locale your_locale`

## Language Codes

Use ISO 639-1 codes:

- **en** - English
- **zh-TW** - Traditional Chinese
- **zh-CN** - Simplified Chinese
- **es** - Spanish
- **fr** - French
- **de** - German
- **ja** - Japanese
- **ko** - Korean
- **pt** - Portuguese
- **ru** - Russian

## Template Translation Tips

When translating template strings:

1. **Preserve placeholders** - Don't translate `{BUILD_DATE}`:
   ```json
   "footer_label": "Built on {BUILD_DATE}"  // ✓ Correct
   "footer_label": "Construido el {BUILD_DATE}"  // ✓ Correct
   ```

2. **Keep string length reasonable** - Avoid very long translations that break layouts

3. **Consider context** - Some words have different meanings:
   - "Light" (in "Light theme") ≠ "lightweight"
   - "Dark" (in "Dark theme") ≠ "sad/gloomy"

4. **Test in browser** - After generating HTML, verify text displays correctly

## Validation

Ensure your JSON is valid:

```bash
# Using Python
python -m json.tool locales/your_locale.json

# Using Node.js
node -e "console.log(JSON.parse(require('fs').readFileSync('locales/your_locale.json')))"
```

## Contributing Translations

To contribute a new language:

1. Create `locales/LOCALE_CODE.json`
2. Translate all strings
3. Test with `python main.py --locale LOCALE_CODE`
4. Submit a pull request to the project

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Language not changing | Check `LOCALE` code matches `locales/LOCALE.json` filename |
| Placeholders not replaced | Verify placeholder format: `{I18N_KEY_NAME}` (uppercase) |
| Text not displaying | Check for JSON syntax errors in your `.json` file |
| Date not showing | Ensure `{BUILD_DATE}` is in `footer_label` string |
| Special characters broken | Save JSON file as UTF-8 encoding |
