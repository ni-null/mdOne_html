Python Socket 觸發機制 — 完整設定文件

架構概覽

Laravel 路由 (web-php-fpm)
    └── Unix Socket
        └── /srv/web/socket/python-socket.sock
            └── systemd socket activation (宿主機)
                └── /srv/web/sh/tasks/python-trigger.sh
                    └── 執行 Python 程式

容器資訊

容器名稱Image說明
web-infrapodman-pausePod 基礎容器
web-redisredis:7.2Redis 快取
web-mariadbmariadb:11.3資料庫
web-apachehttpd:alpineApache Web Server
web-php-fpmlocalhost/web-php-fpm:latestPHP-FPM + Laravel

所有容器共用 web.pod，共享網路。

宿主機設定

1. 建立 Socket 目錄

mkdir -p /srv/web/socket

2. systemd Socket 設定

/etc/systemd/system/python-socket.socket

[Unit]
Description=Python Socket

[Socket]
ListenStream=/srv/web/socket/python-socket.sock
SocketMode=0660
SocketUser=root
SocketGroup=root
Accept=yes

[Install]
WantedBy=sockets.target

⚠️ Accept=yes 是關鍵，每次連線都會 fork 一個新的 service 實例處理請求。

3. systemd Service 設定

/etc/systemd/system/python-socket@.service

⚠️ 注意檔名有 @，配合 Accept=yes 使用。

[Unit]
Description=Python Socket Service
Requires=python-socket.socket

[Service]
Type=simple
ExecStart=/srv/web/sh/tasks/python-trigger.sh
StandardInput=socket
StandardOutput=socket
StandardError=journal

4. Trigger 腳本

/srv/web/sh/tasks/python-trigger.sh

#!/bin/bash
read input
# 在此執行你的 Python 程式
# python3 /path/to/your-script.py
printf "ok\n"
exit 0

chmod 750 /srv/web/sh/tasks/python-trigger.sh

5. 啟動 Socket

systemctl daemon-reload
systemctl enable --now python-socket.socket
systemctl status python-socket.socket

容器設定

php-fpm Quadlet 掛載 Socket

/etc/containers/systemd/web-php-fpm.container 加入：

[Unit]
# 確保 socket 先啟動
After=web.pod web-mariadb.service web-redis.service python-socket.socket
Requires=web.pod python-socket.socket

[Container]
# ... 其他設定 ...

# --- Python Socket ---
Volume=/srv/web/socket/python-socket.sock:/srv/web/socket/python-socket.sock

⚠️ python-socket.socket 必須在容器啟動前已存在，否則 Podman 會將 .sock 建立為普通檔案而非 socket。
若發生此問題，重啟容器即可：systemctl restart web-php-fpm.service

Laravel 路由整合

routes/web.php 或 routes/api.php：

Route::post('/trigger', function () {
    $socket = stream_socket_client(
        'unix:///srv/web/socket/python-socket.sock',
        $errno,
        $errstr,
        5
    );

    if (!$socket) {
        return response()->json(['error' => $errstr], 500);
    }

    stream_set_timeout($socket, 3);
    fwrite($socket, "run\n");
    stream_socket_shutdown($socket, STREAM_SHUT_WR); // 告知對方寫入結束
    $response = trim(stream_get_contents($socket));
    fclose($socket);

    return response()->json(['status' => $response]);
});

⚠️ stream_socket_shutdown($socket, STREAM_SHUT_WR) 是關鍵，沒有這行 PHP 會一直等待，造成請求卡住。

驗證流程

宿主機測試

echo "test" | socat - UNIX-CONNECT:/srv/web/socket/python-socket.sock
# 預期輸出：ok

容器內測試

podman exec -it web-php-fpm php -r "
\$s = stream_socket_client('unix:///srv/web/socket/python-socket.sock', \$e, \$m, 5);
stream_set_timeout(\$s, 3);
fwrite(\$s, 'test\n');
stream_socket_shutdown(\$s, STREAM_SHUT_WR);
echo stream_get_contents(\$s);
fclose(\$s);
"
# 預期輸出：ok

HTTP 測試

curl -X POST https://yourdomain.com/public/trigger
# 預期輸出：{"status":"ok"}

注意事項

項目說明
Accept=yes必須設定，否則 socket 只接受一次連線後就關閉
@ 服務檔名配合 Accept=yes 使用，每次連線 fork 獨立實例
stream_socket_shutdownPHP 端必須呼叫，否則 fread 會永久阻塞
socket 啟動順序容器掛載前 .sock 必須已存在，否則被當普通檔案掛載
After=python-socket.socketQuadlet 加入依賴確保啟動順序正確
SocketMode=0660確認容器內 PHP 程序有讀寫權限
